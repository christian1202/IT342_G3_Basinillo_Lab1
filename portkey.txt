1. Executive Summary
PortKey is an AI-powered customs clearance management platform designed for licensed Customs Brokers and Freight Forwarders in the Philippines. It replaces messy Excel files, scattered WhatsApp threads, and manual BOC tariff book searches with a single, intelligent platform that guides every shipment from vessel arrival to container release.

The platform targets a real, painful problem: Philippine customs brokerage is a high-stakes, time-sensitive industry where a single missed deadline or misclassified product code can cost a client tens of thousands of pesos in fees and fines. PortKey solves this with three core AI-powered features and a real-time shipment tracking dashboard — available on Web, iOS, and Android.

💡 The Core Value Proposition
PortKey does not just manage shipments — it actively prevents the three most expensive mistakes in customs brokerage: demurrage fee overruns, HS Code misclassification penalties, and manual data entry errors.

2. Problem Statement
2.1 Demurrage Doomsday — The Silent Money Drain
Ports charge ₱2,000–₱5,000 per container per day after the free storage window (typically 5–7 days) expires. Brokers today track these critical deadlines in spreadsheets shared via email — no real-time updates, no automated alerts, no urgency indicators. A broker handling 15+ active shipments simultaneously cannot reliably monitor this manually.

2.2 Manual Data Entry — The Fat-Finger Nightmare
A single Commercial Invoice can have 50+ line items. Manually re-keying product descriptions, container numbers, gross weights, and values into the BOC lodgment system is both time-consuming (20+ minutes per document) and error-prone. One incorrect digit in a container number freezes an entire shipment until corrected through a formal amendment process.

2.3 HS Code Misclassification — The Compliance Trap
Every imported good must be assigned an HS Code from the ASEAN Harmonized Tariff Nomenclature (AHTN). The difference between "100% Cotton T-Shirts" (HS 6109.10, 7% duty) and "Polycotton T-Shirts" (HS 6109.90, 15% duty) is not always obvious. Wrong classification triggers BOC penalties, shipment holds, and in extreme cases, smuggling investigations. Currently brokers search a 1,000+ page PDF manually.



3. Shipment Lifecycle (Domain Model)
Every feature in PortKey is built around supporting this 5-stage shipment lifecycle. Understanding this is understanding the entire product.

Stage	Name	Key Event	PortKey System Action
1 — Arrival	Vessel Docks	Broker receives Bill of Lading, Commercial Invoice, Packing List	Shipment created, demurrage clock starts automatically
2 — Lodgment	Entry Filed	Broker files import entry with BOC, declares goods and estimated tax	Entry number logged, docs uploaded to Vault, status updated
3 — Assessment	BOC Review	BOC checks math, assigns Lane: Green / Yellow / Red	Lane status synced, Yellow/Red triggers broker alert
4 — Payment	Duties Paid	Broker pays duties/taxes via authorized bank	OR number recorded, payment confirmed in shipment record
5 — Release	Gate Pass Issued	Gate Pass generated, truck dispatched to collect container	Shipment closed, full audit log finalized, client notified

Golden Documents (Required for Every Shipment)
•	Bill of Lading (BL) — Proof of the shipping contract between shipper and carrier
•	Commercial Invoice (CI) — The seller's bill; declares value and description of goods
•	Packing List (PL) — Itemized manifest of all cargo with weights and dimensions

4. Features
4.1 Core Features
Feature 1 — Demurrage Doomsday Clock
A real-time visual countdown system for every active shipment's free storage window. The dashboard replaces static spreadsheet dates with urgency-driven cards that automatically escalate and alert.

Property	Detail
Trigger	Shipment created with Arrival Date input
Calculation	Free Time Expiry = Arrival Date + Free Days (configurable per shipment, default: 7)
Status: Safe	More than 3 days remaining — Green badge
Status: Warning	2–3 days remaining — Orange pulsing badge + push notification to broker
Status: Critical	Today or overdue — Red flashing badge + SMS + Email alert
Overdue Display	Running ₱ accumulator showing estimated demurrage cost to date
Configuration	Admin can set global default free days; broker can override per shipment
Automation	Cron job runs daily at 8:00 AM to evaluate all active shipments
Alerts Sent To	Assigned broker + org admin via Email (Resend) and Push (Expo + FCM)

Feature 2 — Smart-Scan (AI Document Ingestion)
A drag-and-drop document processor that uses OCR to read a Commercial Invoice or Bill of Lading PDF and automatically pre-fills the shipment entry form. What previously took 20 minutes of manual typing becomes a 10-second "verify and confirm" action.

Step	Action	Technology
1	Broker drags a PDF or image into the upload zone	React Dropzone (Next.js frontend)
2	File uploaded to secure temporary storage	Supabase Storage (free tier)
3	OCR engine processes document and extracts structured data	Google Cloud Document AI or Tesseract OCR
4	NLP layer identifies fields: Consignee, Vessel, Container No., etc.	Spring Boot AI service (Java + LangChain4j)
5	Extracted fields pre-populate the Create Shipment form	Next.js form state (React Hook Form + Zod)
6	Confidence scores shown per field — low confidence highlighted	Frontend UI badges
7	Broker clicks "Verify & Save" to confirm or manually correct	One-click confirmation action

Supported Document Types: PDF, PNG, JPG, TIFF (max 10MB per file)
Extracted Fields: Consignee, Shipper, Vessel Name, Voyage No., Port of Discharge, Container Numbers, Total Gross Weight, Total Value, Currency, Item Descriptions, Invoice Number, Date

Feature 3 — Compliance Copilot (AI HS Code Lookup)
An AI assistant that accepts a plain English product description and returns the best-match HS Code from the Philippine Tariff Schedule, along with the applicable duty rate, VAT, and any special permits required. It transforms a 20-minute manual tariff search into a 5-second confirmation.

Step	Action	Technology
1	Broker types a natural language description: "100% Cotton Men's T-Shirts"	Text input on shipment item form
2	Description is sent to AI backend for classification	Next.js API Route → Spring Boot service
3	RAG pipeline queries the Philippine Tariff Book vector database	LangChain4j + pgvector (NeonDB)
4	LLM returns top 3 HS Code matches with confidence percentage	Google Gemini API (free tier) or OpenAI GPT-4o Mini
5	System displays: HS Code, description, duty rate, VAT, estimated total tax	Frontend result card
6	If item needs a permit, a flag is shown (e.g., Bureau of Standards required)	Rule-based permit checker
7	Broker selects the correct code and it is saved to the shipment line item	Form state update

⚠️ Legal Disclaimer
Compliance Copilot results are AI-assisted estimates only. The final HS Code classification and duty assessment is the sole authority of the Bureau of Customs of the Philippines. PortKey does not guarantee the accuracy of AI-generated tariff classifications.

7. System Architecture
7.1 High-Level Architecture
PortKey follows a decoupled architecture with three primary layers: a Next.js frontend (web), a React Native mobile app, and a Spring Boot backend API — all sharing the same NeonDB PostgreSQL database.

Layer	Technology	Responsibility
Web Client	Next.js 14 → Vercel	Broker-facing UI, Client Portal, SSR pages, API Routes for lightweight operations
Mobile Client	React Native (Expo) → EAS	Broker mobile app, push notifications, mobile document scanning
Backend API	Spring Boot → Railway	Core business logic, auth validation, cron jobs, AI integrations
AI Service	Spring Boot + LangChain4j	RAG pipeline for Compliance Copilot, OCR orchestration for Smart-Scan
Database	NeonDB (PostgreSQL + pgvector)	All persistent data + vector embeddings for HS Code search
File Storage	Supabase Storage	Shipment documents (BL, CI, Packing List, etc.)
Auth Service	Clerk	User identity, org management, JWT issuance, webhooks to sync to DB
Email Service	Resend	Transactional emails — alerts, stage changes, welcome
Push Notifications	Expo FCM + Firebase	Mobile push alerts for demurrage and stage updates

7.2 Data Flow — Smart-Scan
1.	Broker uploads PDF via drag-and-drop in Next.js frontend
2.	File is sent to Supabase Storage; a reference is stored in NeonDB
3.	Next.js API Route calls Spring Boot OCR endpoint with the file URL
4.	Spring Boot calls Google Document AI API; receives structured JSON response
5.	Spring Boot NLP layer maps extracted fields to shipment form schema
6.	Mapped data returned to Next.js; form fields are pre-populated with confidence scores
7.	Broker reviews and clicks Verify & Save — shipment is created in DB

7.3 Data Flow — Compliance Copilot
8.	Broker types product description in the HS Code field
9.	Next.js sends description to Spring Boot /api/copilot endpoint
10.	LangChain4j converts description to vector embedding via OpenAI/Gemini embedding model
11.	Vector is queried against pgvector HS Code database (cosine similarity search)
12.	Top 5 candidate HS Code chunks are retrieved and sent to LLM with the original description
13.	LLM returns top 3 HS Code matches ranked by confidence with duty rate and permit flags
14.	Results displayed in frontend; broker selects and saves to shipment item

7.4 Cron Job — Demurrage Daily Check
15.	Spring Scheduler triggers at 8:00 AM daily (Philippine time: UTC+8)
16.	Query NeonDB: SELECT all shipments WHERE status != RELEASED
17.	For each shipment: compute days_elapsed = TODAY - arrival_date
18.	If days_elapsed >= (free_days - 2): mark as WARNING, trigger email via Resend
19.	If days_elapsed >= free_days: mark as CRITICAL, trigger email + push notification
20.	If days_elapsed > free_days: compute running demurrage cost and update record
21.	All transitions are logged to the AuditLog table

10. Risks & Mitigations
Risk	Impact	Likelihood	Mitigation
BOC tariff regulation changes	Compliance Copilot returns stale HS Codes	Medium	Build admin tool to re-ingest updated tariff PDFs; version the vector database
OCR accuracy on poor-quality scans	Brokers lose trust in Smart-Scan	High	Always show confidence scores; manual override is always the primary path, never blocked
Demurrage miscalculation	Broker liable for financial losses	Low	Free days are configurable per shipment; display the full calculation formula visibly in UI
NeonDB free tier storage limit	App stops working at 0.5GB	Medium	Archive old shipments to cold storage; upgrade to paid plan when first revenue arrives
Gemini / OpenAI API rate limits	AI features slow down or fail	Medium	Implement request queuing, caching of common HS Code lookups, and graceful degradation UI
Multi-tenancy data leak	Catastrophic — clients see each other's shipments	Low	Row-Level Security on NeonDB + Spring Security org-scoped queries on every endpoint
App Store rejection	Delays iOS launch	Low	Follow Apple HIG guidelines from day 1; no background data scraping from BOC
Student time constraints	Delayed delivery	High	Strict MVP scope (MoSCoW); Phase 1 has zero AI dependencies so it ships fast

Commercial Invoice	Seller's official bill to the buyer; used by BOC to determine the taxable value of goods
Packing List	Itemized document listing all contents of a shipment with weights and dimensions
HS Code	Harmonized System Code — a 6–10 digit international classification code for every imported product
Demurrage	Daily fee charged by the port authority when a container is not picked up after the free storage period
Free Days	The number of days a container may sit at the port at no charge, typically 5–7 days
Lane	BOC risk assessment result: Green (auto-release), Yellow (document check), Red (physical examination)
Lodgment	The formal filing of an import entry with the BOC declaring goods, quantities, and estimated duties
Entry Number	BOC-assigned reference number issued upon acceptance of the lodged import entry
Gate Pass	Document issued by the port authority authorizing a truck to collect a specific container
OR Number	Official Receipt number issued by the bank confirming payment of customs duties and taxes
AHTN	ASEAN Harmonized Tariff Nomenclature — the tariff code book used in the Philippines
RAG	Retrieval-Augmented Generation — AI technique that combines a vector database search with an LLM to answer questions from a custom knowledge base
OCR	Optical Character Recognition — technology that reads and extracts text from images and PDFs
pgvector	PostgreSQL extension that enables storing and querying vector embeddings for AI similarity search
MoSCoW	Product prioritization framework: Must Have, Should Have, Could Have, Won't Have (this version)
FCM	Firebase Cloud Messaging — Google's cross-platform push notification delivery service
NeonDB	Serverless PostgreSQL database with a generous free tier, used as PortKey's primary database
